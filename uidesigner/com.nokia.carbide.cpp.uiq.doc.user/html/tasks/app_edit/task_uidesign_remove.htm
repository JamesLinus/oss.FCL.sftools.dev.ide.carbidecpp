Removing a UI Design

When you create a UI design with the UI Designer it is automatically included in the builds of the project to which you originally assign it. If you decide that you do not want that UI Design to be part of your project, you may permanently delete it or temporarily remove it from the project.

Later, if you wish to restore a UI Design which you have temporarily removed - or if you wish to copy a UI Design that was originally created for a different project into your project - you must import the UI Design into the project for it to be included with subsequent builds.

Removing a UI Design

  1. Open the Application Editors UI Design page (Figure 1)

    2. NOTE If the Application Editor is already open, bring it to the front by clicking on its tab.

    Figure 1 -The Application Editor UI Designs page.

    The All UI Designs section of the UI Designs page contains a list of all UI designs currently included in the project.

    The checked item indicates the design displayed when the application is started. It does not have to be the first UI design in the list.

  2. Click the UI design to remove.

    3. The UI Design will be highlighted to indicate that it has been selected.

  3. Click Remove

    4. The UI Design is removed from the UI Design’s list but not from the project.

    NOTE If you remove the checked item, the UI Designer will not check another UI Design for you automatically. You must check the box for the UI Design which you wish to be displayed when the application starts.

    NOTE These changes will not become permanent until your next save of the application.uidesign file.
