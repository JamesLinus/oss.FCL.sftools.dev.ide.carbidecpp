Removing a Language

The Languages page of the Application Editor displays a list of all languages which are currently enabled for your application and indicates which language is displayed in the UI Designer.

Use this page to remove support for any languages which are not required in your application.

Removing a Language from a Project

  1. Select the Languages page of the Application Editor

    2. The Languages page will be displayed.

  2. Select the language to remove in the Included languages listing

    3. The language is highlighted to show it has been selected and the Remove button enabled, as shown in Figure 1.

    Figure 1 - The Languages Page, ready to remove the selected language.

  3. Click Remove

    4. The selected language is removed from the Included languages list.

    NOTE If you remove the checked language (i.e. the language marked for display in the UI Designer) the UI Designer will check the language at the top of the list.

  4. Close the application.uidesign file.

    5. The Save Application Design confirmation dialog appears.

  5. Click the Save button to confirm saving the removal of the language from the application.uidesign file

    6. When you save the application.uidesign file, the UI Designer will remove references to the .lxx files for that language from the .loc files, but it will not delete the .lxx files. This prevents you from losing the localization information you have entered into the .lxx file and thus allows for ease of restoring localization support for the removed language.

    NOTE If you are certain that you will not need the associated .lxx files, you may delete them manually.
