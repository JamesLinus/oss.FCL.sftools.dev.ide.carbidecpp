Container Name and Container Type

Use the Container Name and Type Page of the New S60 UI Design wizard to set the base name and container type for your UI design.

Figure 1 - Setting the Base Name and Container Type

Setting Name and Type

  1. Enter a unique Base name for your UI design or accept the default value
  2. Select the type of container from the Container type context menu or accept the default value
  3. Set the Support view switching checkbox

    4. This option may already be set if you chose to use Avkon View Switching in the project or you are adding the first design to a non-UI Designer project.

    For non-UI Designer projects, UI Designer will not generate the AppUi level code necessary to support view switching regardless of the setting you make on this screen. However, if you leave this box checked it will generate the UI design-specific code necessary to support view switching for this UI design. You must still manually add support for Avkon view switching at the application level within your project code.

    If unchecked, this design will not be included as part of the Avkon view switching mechanism.

  4. Click the Finish button

    5. If your project was already UI Designer enabled, your new UI design will be created and added to the specified project.

    If you are adding a UI design to a project which had not previously been UI Designer enabled, the Project Directories page will appear.
