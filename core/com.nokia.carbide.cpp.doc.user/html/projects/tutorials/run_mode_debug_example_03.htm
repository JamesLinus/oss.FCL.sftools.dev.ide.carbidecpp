Configure project for debugging

In this example we demonstrate how to debug applications on a remote device using the Symbian OS Device (Install SIS) remote debug agent. This enables you to test programs on the actual device it will operate on while observing its behavior using the Carbide debugger. The steps to do this include:

  1. Connect target device to PC
  2. Install CODA remote agent (v3.2.7) or later on the target device
  3. Configure project for downloading
  4. Debug project on target device

 

Configure project for downloading

  1. Select a configuration to debug the HelloWorld application on the target device.

    We choose Phone Debug (armv5_udeb) for this example using the Manage Configurations icon on the toolbar, but yours name may differ based upon the SDK selected for your project.

  2. Click the Debug icon () on the toolbar.

    This opens the New Launch Configuration Wizard dialog asking for additional information. For the first launch you must define the name of the package to install on the target device.

    1. Click the Change... button in the Debug Process section to open the Change Debug Process dialog.

    2. Enable the 'Launch program on device' option and select the executable to launch.

      This tells the CODA remote agent which application to launch once its downloaded.

    3. Click the 'Modify SIS builder settings for build configuration' to open the Properties for project dialog.

      4. The Carbide.c++ > Build Configurations > SIS Builder tab is shown.

    4. Click Add to open the SIS Properties dialog.

    5. Select the project's PKG file to download to the device, then click OK.

    6. Verify the PKG in the SIS Builder tab, then click OK.
  3. Verify the settings in the Change Debug Process, then click OK.

  4. Review the Configure launch configuration setup, then click Debug.

    The project is built, downloaded to the target device, and the executable launched on the device.

On subsequent launches the project will be compiled and downloaded to the target device without any setup on your part. The only time you should need to perform these actions is if you change anything that effects to launch configuration. Of course, you can always modify the settings by using the New or Debug window to edit launch configurations.
